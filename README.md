# WhiteMap

**WhiteMap** — это Android-приложение на Kotlin для отслеживания отключений мобильной связи в Москве и визуализации проблемных зон в реальном времени. Приложение собирает обезличенные данные о сетевом соединении от пользователей и отображает "проблемные зоны" (outage zones) на интерактивной карте.

**Важные ссылки:**
1. запись работы приложения - https://disk.360.yandex.ru/i/7Vq0zBYsNJv_-g
2. APK файл приложения - https://disk.360.yandex.ru/d/KGXsdDmIcaHh9Q

## ⚙️ Принцип работы

1. **Фоновый трекинг:** После первого запуска приложение работает в фоне и регулярно отправляет запросы (пинги) на центральный сервер.
2. **Фиксация сбоев:** Если связь пропадает, приложение записывает текущую геолокацию как offline.
3. **Серверная аналитика:** Сервер ожидает запросов от конкретного пользователя. Если пинги перестают поступать, он записывает последнюю известную геолокацию пользователя как потенциальную offline-зону.
4. **Кластеризация:** Для корректного отображения на карте online и offline точки кластеризуются сервером, после чего готовые зоны транслируются обратно в приложение.

## 🚀 Основные возможности

* **Интерактивная карта:** Использование Yandex MapKit с кастомным минималистичным стилем (удалены лишние POI и административные границы для акцента на данных).
* **Визуализация зон (GeoJSON):** Отображение областей с плохим покрытием в виде полупрозрачных полигонов, подгружаемых с сервера.
* **Фоновый мониторинг:** Специализированный сервис (`ConnectionTrackerService`) продолжает собирать данные о качестве связи, даже если приложение свернуто.
* **Автообновление:** Карта автоматически синхронизируется с сервером каждую минуту.
* **Privacy-First:** Прозрачная система запроса разрешений на геолокацию с объяснением необходимости сбора обезличенных данных.

## 🛠 Технологический стек

* **Язык:** Kotlin
* **UI:** Jetpack Compose (Декларативный интерфейс)
* **Карты:** Yandex MapKit SDK v4
* **Сеть:** Retrofit 2 & Gson (REST API)
* **Архитектура:** Service-oriented, фоновые процессы (Foreground Services)
* **Дизайн:** Material Design 3

## ⚙️ Настройка и установка

1. **API Ключ Яндекса**
Для работы карт необходимо получить ключ в Кабинете разработчика Яндекса. Вставьте ваш ключ в файл `WhiteMapApplication.kt`:
```kotlin
MapKitFactory.setApiKey("ВАШ_КЛЮЧ")

```


2. **Настройка сервера**
По умолчанию приложение обращается к API по адресу `https://med-ai-assistant.ru/`. Изменить адрес можно в `MainActivity.kt` в блоке инициализации Retrofit.
3. **Разрешения**
Приложению требуются следующие разрешения, которые запрашиваются при старте:
* `ACCESS_FINE_LOCATION` / `ACCESS_COARSE_LOCATION` — для определения местоположения.
* `FOREGROUND_SERVICE` / `FOREGROUND_SERVICE_LOCATION` — для работы в фоне.
* `POST_NOTIFICATIONS` — для отображения уведомления о работе сервиса (Android 13+).



## 🖥 Использование

1. **Запуск:** При первом запуске нажмите кнопку "НАЧАТЬ" на приветственном экране и разрешите доступ к геопозиции и уведомлениям.
2. **Просмотр:** На главной вкладке отобразится карта с вашим текущим местоположением и наложенными красными зонами (данные с сервера).
3. **Меню:** Боковое меню позволяет переключаться между картой и разделом "О приложении".
